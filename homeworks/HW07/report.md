## 2. Protocol

Опишем единый протокол "честного" unsupervised‑эксперимента, который применялся ко всем датасетам.

### Препроцессинг

1. **Разделение признаков и идентификаторов**
   - Для всех датасетов из исходных таблиц удаляется колонка `sample_id`.
   - Массив `sample_id` сохраняется отдельно и далее используется только для формирования CSV c метками кластеров.

2. **Масштабирование числовых признаков (обязательно)**
   - Для всех датасетов (A, B, C и дополнительного D) используется `StandardScaler`.
   - Масштабирование выполняется **одинаково для всех моделей** внутри одного датасета, чтобы не было утечки информации и некорректного сравнения алгоритмов.

3. **Обработка пропусков (Dataset 04)**
   - Для числовых признаков Dataset 04 применяется `SimpleImputer(strategy='median')`.
   - Это устойчивый к выбросам способ заполнения пропусков.

4. **Кодирование категориальных признаков (Dataset 04)**
   - Для `cat_a` и `cat_b` используется `OneHotEncoder(handle_unknown='ignore')`.
   - Закодированные признаки конкатенируются с числовыми, далее к ним применяется `StandardScaler`.
   - В результате размерность Dataset 04 после препроцессинга увеличивается до 42 признаков.

5. **Реализация препроцессинга**
   - Для Dataset 04 используется связка `ColumnTransformer` + `Pipeline`, что обеспечивает воспроизводимость и одинаковое преобразование для всех алгоритмов.
   - Для Dataset A/B/C использован более простой пайплайн: `StandardScaler().fit_transform(X)`.

### Поиск гиперпараметров

**KMeans (все датасеты)**

- Диапазон `k`: \[2, 20].
- Фиксированные параметры: `random_state=42`, `n_init=10`, `max_iter` по умолчанию.
- Для каждого `k` считаются метрики:
  - `inertia_` (метод локтя),
  - `silhouette_score`,
  - `davies_bouldin_score`,
  - `calinski_harabasz_score`.
- Основной критерий выбора лучшего `k`: **максимальный Silhouette Score**, при этом графики inertia/DB/CH используются как дополнительная проверка адекватности выбора.

**DBSCAN (на всех датасетах, в т.ч. дополнительном)**

1. **Эвристика для eps: k‑distance plot**
   - Для каждого датасета строится график расстояния до 5‑го ближайшего соседа (отсортирован по возрастанию).
   - На графике визуально выбирается точка излома ("колено"), соответствующая разумному значению `eps`.
   - Рекомендуемое `eps` отображается пунктирной линией и подписью.

2. **Поиск по сетке**
   - Вокруг найденного по k‑distance значения формируется диапазон `eps` (например, ±20–30%).
   - `min_samples` варьируется в небольшом диапазоне (5–10).
   - Для каждой пары `(eps, min_samples)` запускается DBSCAN, считаются метрики, фиксируется доля шума.

3. **Критерий выбора лучшей конфигурации**
   - Основной критерий — **максимальный Silhouette Score** на non‑noise точках.
   - Дополнительные критерии:
     - разумное число кластеров (не сотни мелких);
     - доля шума не близка к 0% *и* не 99+% (см. обсуждение для Dataset 04).

**AgglomerativeClustering (все датасеты)**

- Диапазон `n_clusters`: \[2, 20].
- Методы связи (`linkage`): `'ward'`, `'complete'`, `'average'`.
- Для каждой пары (`k`, `linkage`) считаются silhouette/DB/CH.
- Лучшая конфигурация выбирается по максимальному silhouette при разумном поведении DB и CH.

### Метрики

Для всех моделей и датасетов считаются три внутренние метрики:

- **Silhouette Score** (выше → лучше): соотношение внутрикластерной компактности и межкластерной разделённости.
- **Davies-Bouldin Index (DB)** (ниже → лучше): среднее по кластерам отношение внутрикластерного разброса к расстоянию между центрами кластеров.
- **Calinski-Harabasz Index (CH)** (выше → лучше): отношение межкластерной дисперсии к внутрикластерной.

**Особенность для DBSCAN:**

- Шум точки (`label = -1`) **не участвуют** в расчёте silhouette/DB/CH.
- Отдельно сохраняется и анализируется доля шума (в процентах от всех объектов).

### Визуализация

- **PCA (2D)**: для каждого датасета строятся 2D‑проекции (PC1–PC2) с раскраской по кластерам:
  - отдельные изображения для сравнения KMeans / DBSCAN / Agglomerative;
  - на осях указывается доля объяснённой дисперсии.
- **t‑SNE (опционально)**: построены t‑SNE визуализации для всех датасетов:
  - параметры: `perplexity=30`, `random_state=42`;
  - используются только для **качественного** анализа структуры, без прямой интерпретации расстояний и плотностей.
- **Графики подбора параметров:**
  - `k` vs inertia / silhouette / DB / CH для KMeans;
  - сравнение разных `linkage` для Agglomerative;
  - k‑distance plot для DBSCAN.

## 4. Results

### 4.1 Dataset A

- **Лучший метод и параметры:**
  - **KMeans** с `n_clusters = 2`, `random_state = 42`, `n_init = 10`.
- **Метрики лучшего решения:**
  - Silhouette Score ≈ **0.52**;
  - Davies-Bouldin Index ≈ **0.98**;
  - Calinski-Harabasz Index ≈ **5100**.
- **DBSCAN:**
  - при разумных значениях `eps` и `min_samples` находит схожую двухкластерную структуру, но без явных преимуществ по метрикам и с большей чувствительностью к параметрам.
- **AgglomerativeClustering:**
  - конфигурация `k=2`, `linkage='ward'` даёт практически те же метрики, что и KMeans;
  - это ожидаемо, так как `ward` минимизирует внутрикластерную дисперсию, близко к критерию KMeans.
- **Почему это решение разумно:**
  - после масштабирования кластеры близки к сферическим, чётко разделены по нескольким ключевым признакам;
  - и KMeans, и Agglomerative(ward) дают устойчивое разбиение (см. раздел 5.2), что подтверждает естественность двухкластерной структуры.

### 4.2 Dataset B

- **Лучший метод и параметры:**
  - **AgglomerativeClustering** с `n_clusters = 2`, `linkage = 'average'`.
- **Метрики лучшего решения:**
  - Silhouette Score ≈ **0.42**;
  - Davies-Bouldin Index ≈ **1.0–1.1**;
  - Calinski-Harabasz Index ≈ **3500–4000**.
- **Сравнение с KMeans:**
  - у KMeans silhouette заметно ниже (около 0.25 для типичных значений k);
  - из‑за нелинейной формы кластеров KMeans делит пространство по линейным границам и режет кластеры "пополам".
- **DBSCAN:**
  - способен лучше подхватывать нелинейные структуры и выбросы, но требует тонкой настройки `eps`;
  - при неудачном `eps` либо сливает кластеры, либо маркирует значительную часть точек шумом.
- **Почему это решение разумно:**
  - иерархическая кластеризация с `average` учитывает средние расстояния между кластерами и лучше адаптируется к криволинейной форме;
  - silhouette/DB/CH подтверждают преимущество Agglomerative над KMeans на этом датасете.

### 4.3 Dataset C

- **Лучший метод и параметры (основной выбор):**
  - **KMeans** с `n_clusters = 3`.
- **Метрики для KMeans (k = 3):**
  - Silhouette Score ≈ **0.31** (локальный максимум);
  - Davies-Bouldin Index ≈ **1.1–1.2**;
  - Calinski-Harabasz Index ≈ **6000+**.
- **DBSCAN (лучшая конфигурация):**
  - `eps ≈ 0.36`, `min_samples = 10`;
  - количество кластеров: 3;
  - доля шума ≈ **11–12%**;
  - Silhouette Score ≈ **0.20** (ниже, чем у KMeans).
- **AgglomerativeClustering:**
  - при `linkage='average'` и `k=2` даёт достаточно высокий silhouette, но плохо разделяет разреженные и плотные кластеры;
  - `ward` и `complete` демонстрируют компромисс между числом кластеров и качеством разбиения.
- **Комментарий про DBSCAN:**
  - основная проблема — **разная плотность** кластеров: значение `eps`, подходящее плотным кластерам, делает разреженный кластер частично шумом;
  - увеличение `eps` позволяет захватить разреженный кластер, но начинает сливать плотные.
- **Почему выбор пал на KMeans (k=3):**
  - silhouette/DB/CH указывают на компромиссную, но наиболее стабильную трёхкластерную структуру;
  - в PCA/t‑SNE визуализациях хорошо видны три основные группы, хотя плотности у них различаются.

**Дополнительно: Dataset 04 (опциональный)**

- **Вывод по Dataset 04:**
  - с точки зрения **"классических" метрик кластеризации** (без учёта задачи бизнеса) KMeans/Agglomerative дают более интерпретируемое разбиение (5 кластеров без шума);
  - DBSCAN полезен как инструмент поиска экстремально плотных групп на фоне большого количества более разреженных точек.

## 5. Analysis

### 5.1 Сравнение алгоритмов (важные наблюдения)

1. **Где KMeans "ломается" и почему?**
   - На **Dataset B** с нелинейной структурой KMeans режет кластеры по линейным границам, что приводит к заниженному silhouette по сравнению с Agglomerative.
   - На **Dataset C** с кластерами разной плотности KMeans вынужден балансировать между плотными и разреженными кластерами: часть точек разреженного кластера притягивается к центрам плотных.
   - Во всех датасетах без масштабирования KMeans чувствителен к признакам с большим разбросом → поэтому StandardScaler обязателен.

2. **Где выигрывает DBSCAN / иерархическая кластеризация?**
   - **DBSCAN**:
     - хорошо выявляет плотные кластеры и выбросы, особенно заметно на Dataset 04 (хотя там цена — огромная доля шума);
     - в задачах, где важна устойчивость к выбросам, может быть предпочтительнее KMeans.
   - **AgglomerativeClustering**:
     - на Dataset B (`linkage='average'`) лучше подстраивается под криволинейную форму кластеров;
     - на Dataset A с `linkage='ward'` даёт результаты, сопоставимые с KMeans, но без случайной инициализации центров.

3. **Что сильнее всего влияло на результат?**
   - **Масштабирование**: без StandardScaler silhouette и DB для KMeans и Agglomerative существенно хуже, особенно на Dataset A и Dataset 04.
   - **Дисбаланс плотностей** (Dataset C): сильно осложняет жизнь DBSCAN и частично KMeans.
   - **Высокая размерность и категориальные признаки** (Dataset 04): требуют комбинации imputation + OHE + scaling, иначе алгоритмы "видят" в основном категориальные флаги.

### 5.2 Устойчивость (обязательно для одного датасета)

- **Датасет:** Dataset A (S07-hw-dataset-01).
- **Проверка:** 5 запусков KMeans с `n_clusters = 2` и разными `random_state`.
- **Что делали:**
  - сохраняли метки кластеров для каждого запуска;
  - считали Silhouette Score по каждому запуску;
  - для всех пар разбиений считали Adjusted Rand Index (ARI), далее усредняли.
- **Результаты:**
  - Silhouette Score во всех 5 запусках совпал (≈0.52);
  - ARI mean = 1.0, std = 0.0 (все запуски дают одно и то же разбиение).
- **Вывод:**
  - разбиение на 2 кластера на Dataset A **полностью устойчиво** к случайной инициализации центров;
  - структура данных чётко бимодальна после масштабирования, что подтверждает корректность выбора KMeans(k=2).

### 5.3 Интерпретация кластеров

- **Подход к интерпретации:**
  - для лучшего решения на Dataset A вычислялись средние значения (в стандартизированном масштабе) по каждому признаку в каждом кластере;
  - строился heatmap профилей кластеров по топ‑10 наиболее вариативных признаков.
- **Основные выводы по Dataset A:**
  - кластеры имеют **противоположные профили** по ряду признаков (например, один кластер характеризуется высокими значениями `f01`, `f06`, низкими `f03`, `f04`, в другом — наоборот);
  - часть признаков демонстрирует почти одинаковые средние по кластерам и может рассматриваться как шумовые.
- **Dataset B и C:**
  - интерпретация происходит в основном через 2D‑визуализацию (PCA/t‑SNE), так как признаки синтетические и не имеют прямого бизнес‑смысла;
  - на t‑SNE отчётливо видны раздельные области для кластеров, найденных лучшими моделями.
- **Dataset 04:**
  - из‑за высокой размерности основной инструмент интерпретации — PCA (2D) и распределения по категориям `cat_a`, `cat_b` внутри кластеров KMeans;
  - видно, что некоторые кластеры соответствуют определённым комбинациям категорий и диапазонам числовых признаков.

## 6. Conclusion

Короткие тезисы о том, что было изучено в рамках HW07:

1. **Корректный препроцессинг — основа успешной кластеризации.**  
   Масштабирование числовых признаков, имputation пропусков и кодирование категориальных данных критически важны для distance‑based алгоритмов.

2. **Не существует универсального алгоритма.**  
   KMeans хорошо работает на примерно сферических кластерах (Dataset A), иерархическая кластеризация выигрывает на нелинейных структурах (Dataset B), а DBSCAN полезен для поиска плотных групп и шума (Dataset 04).

3. **Три внутренние метрики дают более полную картину, чем одна.**  
   Silhouette, Davies-Bouldin и Calinski-Harabasz по‑разному реагируют на количество кластеров и форму разбиения, и важно смотреть на них в совокупности.

4. **Визуализация (PCA/t‑SNE) необходима для здравого смысла.**  
   По одним числам метрик сложно оценить адекватность разбиения, тогда как 2D‑проекции позволяют увидеть артефакты (разрезанные кластеры, чрезмерный шум и т.п.).

5. **Устойчивость решений должна проверяться явно.**  
   Даже при фиксированном препроцессинге и алгоритме случайная инициализация (как в KMeans) может привести к разным решениям; ARI даёт удобный способ формализовать устойчивость.

6. **Высокая размерность и категориальные признаки усиливают требования к протоколу.**  
   На Dataset 04 стало очевидно, что без аккуратной обработки пропусков и категориальных столбцов любые выводы о кластерах были бы недостоверны.

7. **DBSCAN требует осмысленного подбора параметров.**  
   Простое "подбором на глаз" значения `eps` редко даёт хороший результат; k‑distance plot и анализ доли шума позволяют сделать выбор более обоснованным.

8. **Unsupervised‑эксперимент должен быть структурирован так же строго, как supervised.**  
   Чёткий протокол (EDA → препроцессинг → систематический подбор параметров → метрики → визуализация → устойчивость → интерпретация) позволяет получать воспроизводимые и объяснимые результаты даже без истинных меток.
